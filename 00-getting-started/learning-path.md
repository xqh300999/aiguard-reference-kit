# 分角色学习路径

> **用途**：本文档为不同角色的开发者提供定制化的学习路径，帮助你在 1 个月（4 个 Sprint）的时间内高效上手并参与项目贡献。实现方式不限，你可根据自身情况调整节奏。

## 通用说明

项目采用 1 个月 4 个 Sprint 的敏捷开发节奏（详见 [01-agile-development](../01-agile-development/)）。每个 Sprint 约 1 周时间。以下路径按 Sprint 划分阶段。

---

## 路径一：后端开发

### 适合人群
- 熟悉 Java/Spring Boot 或其他后端框架
- 了解 RESTful API、数据库、消息队列等概念
- 对物联网/智能家居场景感兴趣

### Sprint 0（准备期，第 0 周）
- [ ] 通读 [00-getting-started](./README.md)，完成环境搭建
- [ ] 学习 [01-agile-development](../01-agile-development/)，理解 Scrum 流程
- [ ] 阅读 [02-project-overview](../02-project-overview/)，理解功能需求
- [ ] 熟悉代码仓库结构，运行后端服务，确认能正常启动
- [ ] 阅读现有代码，了解分层架构（Controller/Service/Mapper/Domain）

### Sprint 1（第 1 周）- 基础功能
- [ ] 认领简单的后端任务（Bug修复、小功能）
- [ ] 熟悉 MQTT 消息流转机制
- [ ] 学习数据持久层设计（MyBatis/JPA 等，实现方式不限）
- [ ] 参与每日站会，熟悉团队节奏
- [ ] 完成至少 1 个功能点的开发与测试

### Sprint 2（第 2 周）- 核心业务
- [ ] 参与核心业务模块开发（设备管理、事件处理、告警规则）
- [ ] 学习规则引擎设计与实现
- [ ] 参与代码评审，理解团队代码规范
- [ ] 编写单元测试与集成测试
- [ ] 对接前端/嵌入式的 API 需求

### Sprint 3（第 3 周）- 高级特性
- [ ] 参与 AI 助手、日报生成等高级功能
- [ ] 学习性能优化与安全加固
- [ ] 参与技术方案讨论
- [ ] 帮助解决其他成员遇到的技术问题

### Sprint 4（第 4 周）- 集成与交付
- [ ] 参与系统集成测试
- [ ] 修复测试中发现的问题
- [ ] 完善文档与部署脚本
- [ ] 参与 Sprint 回顾，总结经验

---

## 路径二：前端开发（Web 面板/移动端）

### 适合人群
- 熟悉 Vue/React/鸿蒙 ArkTS 等前端框架
- 了解组件化开发、状态管理、响应式布局
- 对数据可视化、实时交互感兴趣

### Sprint 0（准备期，第 0 周）
- [ ] 通读 [00-getting-started](./README.md)，完成环境搭建
- [ ] 学习 [01-agile-development](../01-agile-development/)，理解 Scrum 流程
- [ ] 阅读 [02-project-overview](../02-project-overview/)，理解功能需求
- [ ] 运行 Web Dashboard 或鸿蒙 App 项目
- [ ] 熟悉 UI 组件库与项目样式规范

### Sprint 1（第 1 周）- 页面框架
- [ ] 认领简单的页面开发任务
- [ ] 熟悉 API 对接方式与数据模型
- [ ] 实现基础页面布局与组件
- [ ] 学习实时数据推送（SSE/WebSocket/MQTT）
- [ ] 完成至少 1 个页面的开发联调

### Sprint 2（第 2 周）- 核心交互
- [ ] 参与核心功能页面开发（仪表盘、告警、设备管理）
- [ ] 实现数据可视化图表
- [ ] 参与 UI/UX 评审
- [ ] 处理加载态、错误态等边缘情况
- [ ] 对接后端 API，完成联调

### Sprint 3（第 3 周）- 高级交互
- [ ] 参与 AI 助手对话、日报展示等复杂交互
- [ ] 优化页面性能与用户体验
- [ ] 实现动画与过渡效果
- [ ] 适配不同屏幕尺寸

### Sprint 4（第 4 周）- 集成与交付
- [ ] 参与端到端测试
- [ ] 修复 UI 问题与交互 Bug
- [ ] 完善前端文档
- [ ] 参与 Sprint 回顾

## 路径三：嵌入式开发（固件/硬件）

### 适合人群
- 熟悉 ESP32/嵌入式 C/C++ 开发
- 了解 RTOS、外设驱动、网络通信
- 对物联网硬件、智能设备感兴趣

### Sprint 0（准备期，第 0 周）
- [ ] 通读 [00-getting-started](./README.md)，完成环境搭建
- [ ] 学习 [01-agile-development](../01-agile-development/)，理解 Scrum 流程
- [ ] 阅读 [02-project-overview](../02-project-overview/)，理解功能需求
- [ ] 搭建 ESP-IDF 或 Arduino 开发环境
- [ ] 编译并烧录现有固件，确认硬件正常工作

### Sprint 1（第 1 周）- 基础驱动
- [ ] 认领简单的固件任务（驱动调试、小功能）
- [ ] 熟悉硬件原理图与引脚定义
- [ ] 学习 MQTT 客户端实现与消息协议
- [ ] 实现传感器数据采集
- [ ] 完成至少 1 个传感器/外设的驱动调试

### Sprint 2（第 2 周）- 设备功能
- [ ] 参与屏幕 UI 或手表交互功能开发
- [ ] 实现 WiFi 配网与连接管理
- [ ] 学习低功耗优化
- [ ] 实现告警逻辑与本地提示
- [ ] 与后端联调 MQTT 通信

### Sprint 3（第 3 周）- 高级功能
- [ ] 参与小智 AI 语音交互集成
- [ ] 实现运动检测、手势识别等算法
- [ ] OTA 升级功能
- [ ] 硬件稳定性测试与优化

### Sprint 4（第 4 周）- 集成与交付
- [ ] 参与硬件集成测试
- [ ] 修复设备端 Bug
- [ ] 完善固件文档与烧录说明
- [ ] 参与 Sprint 回顾

## 路径四：全栈开发

### 适合人群
- 具备前后端或嵌入式+云端的多领域开发能力
- 希望理解系统全貌，能够独立完成端到端功能
- 未来想成为技术负责人或架构师

### 学习建议
1. **先广后深**：先了解各层技术栈，再选择 1-2 个方向深入
2. **端到端实践**：尝试独立完成一个小功能从设备到云端到前端的全链路
3. **关注架构**：多思考模块边界、接口设计、数据流转
4. **按 Sprint 节奏**：参考后端 + 前端/嵌入式路径，但每个 Sprint 任务量适当调整

### Sprint 0（准备期）
- [ ] 同其他角色，额外增加：通读全部项目文档
- [ ] 跑通从设备/模拟设备 → 后端 → 前端的完整链路
- [ ] 画一张自己理解的系统架构图

### Sprint 1-4
- 在其他角色路径基础上，每个 Sprint 额外承担：
- [ ] 至少一个端到端功能的设计与实现
- [ ] 参与接口设计讨论，关注跨模块协作
- [ ] 帮助团队定位跨层级问题

## 路径五：项目经理 / Scrum Master

### 适合人群
- 负责项目进度、团队协作、流程改进
- 不需要深入代码，但需要理解技术可行性
- 擅长沟通协调、风险识别

### Sprint 0（准备期）
- [ ] 通读所有入门与敏捷文档
- [ ] 理解项目目标与范围（02-project-overview）
- [ ] 熟悉团队成员角色与能力
- [ ] 搭建项目管理工具（看板、燃尽图等）

### Sprint 1-4
- [ ] 组织并主持所有 Scrum 仪式（计划会、站会、评审、回顾）
- [ ] 跟踪任务进度，识别并解决阻塞
- [ ] 维护产品待办列表（Product Backlog）
- [ ] 促进团队沟通与协作
- [ ] 持续改进团队流程

---

## 通用建议

1. **不要害怕提问**：遇到问题先尝试自己解决 15-30 分钟，然后大胆提问
2. **小步提交**：频繁提交小的代码变更，降低集成风险
3. **代码评审**：认真对待代码评审意见，也积极评审他人代码
4. **文档同步**：代码变更时同步更新相关文档
5. **劳逸结合**：保持可持续的开发节奏，避免加班冲刺
6. **鼓励创新**：如果你有更好的实现想法，大胆提出来讨论！

> **提示**：以上路径仅为参考。实现方式不限，你可以根据自己的兴趣和项目实际需求灵活调整。核心目标是在 4 个 Sprint 内从入门成长为能够独立贡献的团队成员。
