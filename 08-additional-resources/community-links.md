# 社区论坛链接

本文档整理相关技术社区和论坛，遇到问题时可以在这些社区搜索答案或提问交流。提问前请先阅读社区规则，学会正确提问。

---

## 一、通用编程社区

### Stack Overflow
- 地址：https://stackoverflow.com/
- 说明：全球最大的程序员问答社区，几乎所有编程问题都能在这里找到答案
- 使用建议：
  - 提问前先搜索，99%的问题已经有人问过
  - 搜索时用英文关键词，结果更丰富
  - 如果提问，提供最小可复现代码、完整错误信息、你尝试过的方法
  - 遵守社区规则，问题要具体、清晰

### GitHub
- 地址：https://github.com/
- 说明：全球最大的开源代码托管平台
- 使用建议：
  - 使用项目的Issues功能反馈bug
  - 提问前先看项目的README、文档、已有Issues
  - 可以查看优秀开源项目的源码学习
  - 提交Issue时按照模板提供完整信息

### GitChat
- 地址：https://gitbook.cn/
- 说明：国内IT技术社区，有专栏、问答、课程

### 开源中国 (OSCHINA)
- 地址：https://www.oschina.net/
- 说明：国内开源技术社区，有项目、资讯、问答

## 六、如何正确提问

在社区提问时，好的提问方式更容易得到准确回答，也节省大家的时间。

### 提问前

1. **先搜索**：用搜索引擎搜索你的问题，包括错误信息、问题关键词，90%的问题已经有答案
2. **看文档**：先查阅官方文档和入门教程，确认不是基础问题
3. **自己尝试**：按照故障排查指南自己排查一遍，记录下你尝试过的步骤和结果
4. **看已有问题**：在论坛/Issues里搜索，看有没有人遇到同样的问题

### 提问时

1. **清晰明确的标题**：一句话概括问题，不要写"求助""帮帮我"这类标题
   - 错误示例："救命啊运行报错了"
   - 正确示例："ESP32连接WiFi失败，错误码ESP_ERR_WIFI_PASSWORD，密码确认正确"

2. **环境信息**：
   - 操作系统和版本
   - 相关软件/库/SDK版本
   - 硬件型号（如果是硬件问题）

3. **详细描述问题**：
   - 你做了什么操作？
   - 期望得到什么结果？
   - 实际出现了什么现象？
   - 问题是必现还是偶现？什么情况下出现？

4. **错误信息**：
   - 完整的错误日志和堆栈信息，不要只截取一部分
   - 用代码块格式包裹日志，方便阅读
   - 如果有截图，确保关键信息清晰

5. **你尝试过的方法**：
   - 你已经搜过哪些关键词？
   - 你尝试过哪些解决方法？结果如何？
   - 不要让别人帮你试你已经试过的方法

6. **最小复现**：
   - 如果是代码问题，提供最小可复现代码，而不是整个项目
   - 说明复现步骤

### 提问后

1. **及时反馈**：如果有人回答，尝试后及时反馈结果
2. **解决后总结**：问题解决后，可以总结一下解决方案，方便后来人遇到同样问题时参考
3. **礼貌尊重**：社区里回答问题的人都是义务帮忙，保持礼貌

## 社区参与建议

1. **先潜水，再发言**：刚加入社区时先看一段时间，了解社区文化和提问方式
2. **不仅索取，也要贡献**：问题解决后可以分享经验，也可以回答自己懂的问题
3. **尊重知识产权**：引用别人的文章/代码注明出处，不要直接抄袭
4. **理性讨论**：技术问题不要人身攻击，客观讨论问题
5. **注意安全**：不要在公开场合泄露密钥、密码、内网地址等敏感信息
