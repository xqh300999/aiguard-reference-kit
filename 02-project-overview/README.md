# 项目概览

> **用途**：本文档是项目概览模块的目录说明。本模块只定义 AiGuard 项目的功能目标、需求范围和参考架构，**不限制具体实现方式**，鼓励团队创新和探索更优方案。

## 模块概述

本模块描述 AiGuard 智能家居养老看护系统「要做什么」，而不是「怎么做」。功能需求定义了系统需要达成的目标，参考架构提供了一种可行的思路，非功能需求设定了质量标准。所有这些都是指导和参考，而非约束。

## 重要声明

⚠️ **本模块内容只提供功能目标，不限制实现！**

- ✅ 你可以选择任何合适的技术栈
- ✅ 你可以采用不同的架构设计
- ✅ 你可以调整功能优先级
- ✅ 你可以引入新的特性和创新
- ❌ 不要被参考实现所束缚
- ❌ 不要认为只有一种正确的实现方式
- 💡 鼓励探索、尝试、创新

## 功能目标一句话总结

AiGuard 是一套面向居家养老场景的智能看护系统，通过传感器、智能设备、AI 助手等技术手段，为老年人提供安全守护、健康监测、紧急求助、亲情关怀等服务，同时为家属和护理人员提供远程看护能力。

## 建议阅读顺序

1. 阅读本 README 理解模块定位
2. 阅读 [functional-requirements.md](functional-requirements.md) 了解系统要实现哪些功能
3. 浏览 [system-architecture.md](system-architecture.md) 了解参考架构（记住：只是参考！）
4. 阅读 [non-functional-requirements.md](non-functional-requirements.md) 了解质量要求
5. 开始设计和实现——选择你认为最合适的方案！

## 给开发者的话

> "优秀的实现不是因为遵循了模板，而是因为真正理解了问题，并用最合适的方式解决了它。"

参考资料是拐杖，不是枷锁。当你理解了需求之后，大胆用你认为最好的方式去实现。如果你的方案比参考方案更好，那正是我们希望看到的。
