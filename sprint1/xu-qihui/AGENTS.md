# 智守项目 AI 行为规则

## 核心原则
- 先思考，再编码：接到任务先分析需求，确认后再动手
- 简单优先：能用 50 行解决的问题不用 200 行
- 精准修改：只改需求涉及的行，不顺手重构相邻代码
- 目标驱动：每段代码都要有明确的目标，不写"以防万一"的功能

## 技术栈约束
- 后端：Java/SpringBoot，遵循现有包结构
- 前端：Vue 3 + TypeScript
- AI 服务：Python/FastAPI
- 硬件：ESP32 + C
- 数据库：使用项目已有的数据库，不引入新依赖

## 代码规范
- 先写测试，再写实现（TDD）
- 测试覆盖率不低于 70%
- API 设计遵循 RESTful 风格
- 错误信息用中文返回
- 不使用不存在的库或 API——如果不确定，先查文档再写
