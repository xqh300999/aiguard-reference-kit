# Windows上生成密钥公钥（7.19）
今天在 Windows 上配置 树莓派 设备的 SSH 免密登录，记录一下整个过程。

---

## 背景
需要从 Windows 电脑免密登录 树莓派 设备（IP：192.168.3.19，用户名：rapi）。

已经配置了基础 SSH 连接，但每次都要输密码太麻烦，目标是实现免密登录。

---

## 第一步：检查本地是否有密钥
先看看自己 Windows 电脑上有没有现成的密钥：

```bash
dir C:\Users\szpu\.ssh\
```

结果显示：

```plain
2026-07-09  14:23    <DIR>          .
2026-07-09  14:23    <DIR>          ..
2026-07-09  14:23             1,822 id_ed25519
2026-07-09  14:23               438 id_ed25519.pub
```

发现已经有 `id_rsa` 和 `id_rsa.pub` 了，省去生成步骤。

> 如果没有密钥的话，用 `ssh-keygen -t ed25519` 生成即可，一路按回车。
>

---

## 第二步：查看并复制公钥内容
查看公钥内容：

```bash
type C:\Users\szpu\.ssh\id_ed25519.pub
```

或者用更通用的方式（如果用 Git Bash 的话）：

```bash
cat ~/.ssh/id_ed25519.pub
```

屏幕上显示：

```plain
ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQC...（省略具体内容）... szpu@517_stu24
```

复制了那串以 `ssh-id_ed25519`开头的完整文本。

---

## 第三步：尝试 ssh-copy-id（踩坑）
在 Windows PowerShell 里执行 `ssh-copy-id`：

```powershell
PS C:\Users\szpu> ssh-copy-id rapi@192.168.3.19
```

结果报错：

```powershell
ssh-copy-id : 无法将"ssh-copy-id"项识别为 cmdlet、函数、脚本文件或可运行程序的名称
```

这才发现 **Windows 默认没有 **`ssh-copy-id`** 命令**。

---

## 第四步：手动部署公钥（解决方案）
既然没有 `ssh-copy-id`，那就手动来。

先用密码方式登录远程 树莓派 设备：

```bash
ssh rapi@192.168.3.19
```

输入密码，登录成功。

在远程设备上创建 `.ssh` 目录，并把公钥写入 `authorized_keys`：

```bash
mkdir -p ~/.ssh
echo "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQC..." >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
chmod 700 ~/.ssh
```

---

## 第五步：测试免密登录
退出远程连接：

```bash
exit
```

重新连接测试：

```bash
ssh rapi@192.168.3.19
```

这次没有问密码，直接进去了！

---

## 完整命令（一次性执行）
为了方便以后快速配置，记录一下完整命令（在远程设备上执行）：

```bash
mkdir -p ~/.ssh && echo "ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQC..." >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys && chmod 700 ~/.ssh
```

---

## 总结与关键点
> **关键点：**
>
> 1. 公钥在 **Windows 本地**生成，私钥留在本地，公钥放到远程服务器。
> 2. Windows 没有 `ssh-copy-id` 命令，需要**手动 **`echo`** 追加**到 `~/.ssh/authorized_keys`。
> 3. 别忘了设置权限：`~/.ssh` 目录权限为 `700`，`authorized_keys` 文件权限为 `600`。
> 4. 多个公钥可以共存，每行一个，追加到 `authorized_keys` 文件即可。
>

---

**搞定收工！** ✅

# 安装Home Assistant
使用CMD连接树莓派

```bash
ssh rapi@192.168.3.19
```

### 准备工作
1. **确保系统就绪**：你的树莓派5已安装好Raspberry Pi OS（或其他64位系统），并连接了网络。
2. **安装Docker**：最快捷的方式是使用官方安装脚本。在终端执行：

```bash
curl -sSL https://get.docker.com | sh
```

安装完成后，将当前用户加入docker组以避免每次使用`sudo`（需要重新登录生效）：

```bash
sudo usermod -aG docker $USER
```

然后注销并重新登录，或重启树莓派。

### 使用 Docker Compose 安装 Home Assistant
这是官方推荐的方式，它通过一个YAML文件定义所有配置，方便日后管理和升级。

1. **创建项目目录**：首先，为Home Assistant创建一个文件夹，用于存放配置文件和Docker Compose文件。

```bash
mkdir -p ~/homeassistant
cd ~/homeassistant
```

2. **创建 **`docker-compose.yml`** 文件**：使用`nano`编辑器创建该文件。

```bash
nano docker-compose.yml
```

3. **粘贴以下内容到文件中**：

```yaml
version: '3'
services:
  homeassistant:
    # 这是专为树莓派5优化的64位镜像，性能更好
    image: ghcr.io/home-assistant/raspberrypi5-64-homeassistant:stable
    container_name: homeassistant
    restart: unless-stopped
    privileged: true
    network_mode: host
    environment:
      - TZ=Asia/Shanghai # 请将此处时区替换为你所在时区
    volumes:
      - ./config:/config
```

> **关键配置项说明**：
>
> + `image: ghcr.io/home-assistant/raspberrypi5-64-homeassistant:stable`：这是Home Assistant官方为树莓派5 64位系统提供的专用镜像，能充分利用硬件性能。如果你使用其他系统，也可以换成 `ghcr.io/home-assistant/home-assistant:stable`。
> + `network_mode: host`：使用主机网络模式，可以让Home Assistant自动发现局域网内的设备，简化网络配置。
> + `privileged: true`：赋予容器更高权限，部分硬件集成（如蓝牙）可能需要此设置。
> + `volumes: - ./config:/config`：将容器内的配置文件映射到树莓派当前目录下的`config`文件夹，这样配置不会随容器更新而丢失。
>

按 `Ctrl+X`，然后按 `Y`，再按 `Enter` 保存并退出。

4. **启动容器**：在 `~/homeassistant` 目录下执行以下命令，Docker Compose会自动拉取镜像并启动服务。

```bash
docker compose up -d
```

### 访问与配置
1. **查看运行状态**：

```bash
docker compose ps
```

你应该能看到状态为 `Up`。

2. **访问 Home Assistant**：在你的电脑浏览器中输入：

```plain
http://192.168.3.19:8123
```

（请将 `<你的树莓派IP地址>` 替换为实际的IP，例如 `http://192.168.1.100:8123`）

3. **初始化设置**：按照页面指引创建账户、设置位置等信息，即可进入Home Assistant仪表盘。

### 后续建议
+ **安装HACS（Home Assistant Community Store）**：这是一个非常流行的第三方插件商店，可以极大丰富Home Assistant的功能。如果后续需要安装，可以参考HACS官方文档中关于Docker部署的指导，基本步骤是进入容器并执行安装脚本。
+ **日志查看**：如果启动过程中遇到问题，可以通过以下命令查看详细日志来排查：

```bash
docker compose logs -f
```

**用Docker安装MQTT**

### <font style="color:rgb(15, 17, 21);">使用 Docker 自动管理的卷（最简单，直接复制全部）</font>
```bash
docker run -d \
  --name=mosquitto \
  --restart=always \
  -p 1883:1883 \
  -p 9001:9001 \
  -v mosquitto_config:/mosquitto/config \
  -v mosquitto_data:/mosquitto/data \
  -v mosquitto_log:/mosquitto/log \
  eclipse-mosquitto
```

出问题了：

```latex
Unable to find image 'eclipse-mosquitto:latest' locally docker: Error response from daemon: failed to resolve reference "docker.io/library/eclipse-mosquitto:latest": failed to do request: Head "https://docker.mirrors.ustc.edu.cn/v2/library/eclipse-mosquitto/manifests/latest?ns=docker.io": dial tcp: lookup docker.mirrors.ustc.edu.cn on 192.168.3.1:53: no such host
```

> <font style="color:rgb(15, 17, 21);">问题找到了！你的树莓派配置了中科大的 Docker 镜像加速器（</font>`<font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">docker.mirrors.ustc.edu.cn</font>`<font style="color:rgb(15, 17, 21);">），但这个镜像源目前</font>**<font style="color:rgb(15, 17, 21);">已经停止服务</font>**<font style="color:rgb(15, 17, 21);">了，导致无法拉取镜像。</font>
>
> <font style="color:rgb(15, 17, 21);">我们需要换一个可用的镜像源，或者直接使用 Docker 官方源。</font>
