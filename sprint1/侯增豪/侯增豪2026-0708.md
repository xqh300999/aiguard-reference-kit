# 2026-0708

今天任务是：把涂鸦SOS报警器通过HA接到AiGuard后端。听起来很明确对吧，结果一整天都在坑里爬。

## 上午

首先确认环境：
+ 树莓派5刷了HA OS，无屏幕，只能SSH/浏览器访问
+ 树莓派IP：`192.168.3.11`
+ 电脑IP：`192.168.3.15`
+ HA Web界面已打开，创建了管理员账号

用`ipconfig`找到路由器网关是`192.168.3.1`，登进路由器后台找到了树莓派的IP。
> 也是顺利的

SSH连树莓派：`ssh pi@192.168.3.11`
报错：
```
ssh: connect to host 192.168.3.11 port 22: Connection refused
```
懵圈了。查了一下，HA OS没有默认的`pi`账号，SSH服务默认不开启。
+ 方案一：用HA的Web终端插件 → 有效
+ 方案二：在SD卡boot分区新建`ssh`空文件 → 没试，因为方案一已经能用了

在HA左侧边栏→配置→加载项→加载项商店→搜索"Terminal & SSH"→安装启动。在浏览器里就有了终端，不用折腾SSH了。

## 涂鸦集成

在HA里添加涂鸦集成，用的是"Tuya"官方插件，输入用户代码后成功添加了SOS报警器。
> 这一步其实还算顺利

在"设备与服务"里能看到SOS按钮设备，实体列表里有`binary_sensor.xxx_emergency`之类的。
