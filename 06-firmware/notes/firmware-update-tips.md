# 固件升级/回退注意事项

## 升级前准备

### 1. 确认兼容性
- 确认新固件支持你的硬件型号（esp32s3-watch vs esp32p4-panel）
- 阅读版本发布说明（Release Notes），了解变更内容和已知问题
- 注意是否有分区表变更（分区表变更必须擦除Flash重新刷写）
- 确认从当前版本升级到目标版本是否有特殊要求（如必须先升级到中间版本）

### 2. 备份（如可能）
- 记录当前Wi-Fi配置（升级通常不会丢失，但OTA回退可能会）
- 备份重要配置（MQTT服务器地址、紧急联系人等）
- 如果设备有重要数据在Flash/SD卡，先备份
- 如使用esptool.py，可读取Flash做完整备份：
  ```bash
  esptool.py --chip esp32s3 --port PORT read_flash 0x0 0x200000 backup.bin
  ```

### 3. 确保供电稳定
- 使用稳定的USB电源/端口
- 笔记本电脑确保电量充足，避免升级中途断电
- 电池供电的设备建议在充电状态下升级
- OTA升级时确保Wi-Fi信号良好

### 4. 选择合适时间
- 预留10-30分钟完成升级和验证
- 避免在紧急情况下升级
- OTA升级避免在网络高峰期

## 刷写升级（USB）

### 必须擦除Flash的情况
- 分区表发生变更（不同版本之间分区布局不同）
- 刷写后无法启动、反复重启
- 从其他固件切换到小智固件
- 降级到大版本更新之前的版本
- 出现奇怪的运行时错误

```bash
esptool.py --chip esp32s3 --port PORT erase_flash
```

### 保留数据的升级（OTA或双分区）
- OTA升级使用双分区方案（ota_0/ota_1），无需擦除
- USB双分区刷写时如分区表未变，只刷写app分区即可保留配置
- 但NVS中存储的配置如果有结构变更，可能需要重置

## OTA 无线升级注意事项

### OTA升级流程
1. 设备连接Wi-Fi
2. 检查新版本
3. 下载固件到空闲OTA分区
4. 校验固件完整性
5. 设置启动分区为新固件
6. 重启进入新固件
7. 确认新固件运行正常
8. 如异常，自动回退或手动回退

### OTA升级风险
- **断电风险**：下载完成前断电通常无影响，重启继续使用旧固件；写入新分区后重启过程中断电可能导致启动失败
- **网络中断**：HTTP支持断点续传，中断后可继续；如服务器不支持断点续传需重新下载
- **校验失败**：固件损坏或传输错误，设备不会切换到坏固件
- **空间不足**：Flash空间不足时OTA会失败，不会影响现有固件

### OTA升级建议
- 使用稳定的Wi-Fi连接，避免在信号弱的地方OTA
- 升级过程中不要断电或重启设备
- 确保电池电量 > 50% 或连接电源
- 从可信来源获取固件，校验MD5/SHA256
- 重要设备先在测试设备上验证新固件

## 固件回退

### OTA回退
如果新固件有问题，可以回退到之前的版本：
1. 设备支持自动回退：新固件启动后如未标记成功，下次启动自动回退
2. 手动触发回退（如果固件提供此功能）：通过菜单或命令设置
3. 回退机制取决于固件实现，v2.2.4请参考发布说明

### USB回退
- 下载旧版本固件
- 必要时擦除Flash（配置不兼容时建议擦除）
- 按正常刷写流程刷入旧版本固件

## 刷写后验证清单

刷写/升级完成后按以下步骤验证：

### 1. 基础启动
- [ ] 设备重启，屏幕正常点亮
- [ ] 无重启循环（串口日志无反复重启）
- [ ] 显示开机画面/主界面

### 2. 系统功能
- [ ] 按键响应正常
- [ ] 触摸操作正常（面板/手表）
- [ ] 屏幕显示无异常（花屏、闪烁等）
- [ ] 音频播放正常（如有）
- [ ] 麦克风正常（如有）

### 3. 连接功能
- [ ] Wi-Fi正常连接
- [ ] OTA检查正常
- [ ] MQTT连接正常（如配置）
- [ ] 小智AI对话正常（如使用）

### 4. AIguard功能
- [ ] IMU传感器数据正常读取
- [ ] 跌倒检测功能可用（可测试轻拍/甩动模拟）
- [ ] 电池电量显示正常
- [ ] 告警功能正常（测试告警上报）

### 5. 长期稳定性（观察）
- [ ] 运行1小时无异常重启
- [ ] Wi-Fi不会频繁断开
- [ ] 内存无明显泄漏
- [ ] 待机功耗/发热正常

## 常见问题处理

### 刷写后黑屏/无显示
1. 按RESET键复位一次
2. 确认固件对应正确的硬件型号
3. 通过串口日志查看启动信息
4. 检查屏幕初始化是否成功（可能固件版本不匹配屏幕驱动）
5. 重新刷写，确认刷写过程无错误
6. 擦除Flash后刷写其他版本/出厂固件

### 刷写后反复重启（Bootloop）
可能原因：
- **分区表错误**：刷写了错误的分区表，重新刷写对应版本的分区表
- **Flash大小配置错误**：确保flash_size设置为32MB
- **固件损坏**：重新下载固件，校验MD5后刷写
- **分区表不兼容**：erase_flash后重新刷写完整固件
- **PSRAM配置错误**：S3需要启用OPI PSRAM

### OTA后无法启动
1. 尝试手动按RESET键几次
2. 如固件支持自动回退，等待几秒应自动回退到旧版
3. 无法回退时，使用USB刷写恢复
4. 下次OTA前确认固件完整性

### 配置丢失
- 擦除Flash后所有配置都会丢失，需要重新配网
- OTA升级通常保留配置
- NVS区域损坏也会导致配置丢失，可擦除Flash重刷
- 建议刷写前记录关键配置

### 无法进入下载模式
参见各硬件flashing-guide.md的进入下载模式章节。

## 版本回退策略建议

- 升级前保留一份已知稳定的旧版本固件文件
- 保留checksums.txt中的校验值，防止固件文件损坏
- 重大版本升级前先刷到测试设备验证
- 生产环境设备不要立即更新到刚发布的新版本，等待稳定期
- 保留至少一个可USB刷写的恢复方案（即使OTA为主）
