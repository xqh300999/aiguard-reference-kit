# 前端问题排查

前端包括Web管理界面和HarmonyOS应用。在排查前端问题前，请确保环境、网络、后端服务都已验证正常。

## 诊断前自查清单
- Node.js、npm/yarn/pnpm版本是否符合要求？
- 后端服务是否正常启动？能否直接访问后端API？
- 浏览器控制台是否有错误信息？按F12查看
- 前端依赖是否完整安装？node_modules是否存在？
- 你是在开发模式还是生产构建模式下遇到问题？

---

## 一、构建失败问题

### 现象特征
- `npm run build`/`yarn build`报错
- 开发服务器启动失败
- 编译过程中出现语法错误、模块找不到
- 构建产物不完整或无法运行

### 引导式排查

1. **依赖安装验证**
   - node_modules目录是否存在？是否完整？
   - 删除node_modules和lock文件，重新安装依赖试过吗？
   - package.json中的依赖版本是否有冲突？
   - 是否在正确的目录下执行构建命令？
   - 全局安装的包是否和本地版本冲突？

2. **语法与类型错误**
   - 错误信息指向哪个文件哪一行？
   - 是JavaScript/TypeScript语法错误吗？
   - 是否使用了项目Node版本不支持的语法？
   - TypeScript类型错误具体是什么？是类型定义缺失还是类型不匹配？
   - 最近修改了哪些文件？错误是否在你修改之后出现？

3. **资源加载问题**
   - 是否有静态资源（图片、字体、样式文件）找不到？
   - 资源路径是相对路径还是绝对路径？是否正确？
   - 文件扩展名大小写是否正确？（Windows下不敏感但Linux下敏感）
   - 导入路径是否有拼写错误？
   - 是否配置了路径别名（如@/）？别名配置是否正确？

4. **构建配置问题**
   - 构建工具配置文件（vite.config.ts/webpack.config.js等）是否被修改过？
   - 环境变量文件（.env.development/.env.production）是否存在且配置正确？
   - 构建输出目录是否有写入权限？
   - 是否有插件版本和构建工具版本不兼容？
   - 构建内存是否不足？（大型项目可能需要增加Node内存限制）

### 验证方法
- 查看构建日志的第一个错误，通常第一个错误是根源，后续错误是连锁反应
- 回退到上次能正常构建的代码版本，对比差异
- 尝试新建一个最小项目，引入出问题的模块，验证是否是模块本身的问题

## 二、跨域（CORS）问题

### 现象特征
- 浏览器控制台出现CORS相关错误
- API请求失败，Network标签中显示CORS错误
- 预检请求（OPTIONS）失败
- 开发环境正常但生产环境跨域

### 引导式排查

1. **错误识别**
   - 控制台具体的CORS错误信息是什么？
   - 是"No 'Access-Control-Allow-Origin' header"错误吗？
   - 是预检请求失败吗？（OPTIONS请求返回非2xx状态码）
   - 是允许的方法/头部不匹配吗？
   - 是凭证请求（withCredentials）但配置不匹配吗？

2. **后端配置验证**
   - 后端是否配置了CORS中间件？
   - CORS配置中允许的Origin是否包含你前端的访问地址？
   - 注意Origin要精确匹配：协议（http/https）+ 域名/IP + 端口，缺一不可
   - 允许的HTTP方法是否包含你使用的方法（GET/POST/PUT/DELETE等）？
   - 允许的请求头是否包含你发送的自定义头部？
   - 如果请求带Cookie/Authorization，是否配置了credentials: true？
   - 是否配置了正确的预检请求缓存时间？

3. **开发环境代理配置**
   - 开发服务器的代理配置是否正确？（vite的proxy、webpack的devServer.proxy）
   - 代理的目标地址是否正确？是否能访问？
   - API请求的路径是否匹配代理规则？
   - 是否配置了changeOrigin: true？
   - WebSocket代理是否单独配置？

4. **生产环境部署问题**
   - 生产环境前端和后端是否同源部署？（反向代理）
   - Nginx/Apache等反向代理是否正确配置了CORS头？
   - 是否有CDN或WAF修改了响应头？
   - HTTPS站点访问HTTP接口也会被浏览器拦截，确认协议一致

### 验证方法
- 在Network标签中查看OPTIONS请求的响应，看后端返回了哪些CORS头
- 使用curl模拟OPTIONS预检请求，对比和浏览器发送的请求有什么不同
- 临时在后端设置允许所有Origin（*），确认是否是CORS配置问题（仅限调试）

## 三、接口请求问题

### 现象特征
- API请求返回4xx/5xx错误
- 请求超时无响应
- 请求发送了但参数不对
- 返回数据格式异常导致前端报错
- 登录状态/Token失效

### 引导式排查

1. **请求基本信息**
   - 在浏览器Network标签中找到该请求：
     - 请求URL是否正确？有没有拼写错误？
     - 请求方法（GET/POST）是否正确？
     - 请求状态码是多少？（400/401/403/404/500/502等）
   - 这个接口通过curl/Postman直接调用是否正常？

2. **请求参数问题**
   - 请求参数是放在query、body还是path中？位置是否正确？
   - GET请求参数是否正确拼接到URL上？
   - POST请求的Content-Type是否正确？（application/json、application/x-www-form-urlencoded、multipart/form-data）
   - 请求体格式是否和Content-Type匹配？
   - 参数名称是否拼写正确？大小写是否正确？
   - 参数值类型是否正确？（数字vs字符串、布尔值格式）

3. **认证授权问题**
   - 如果是401 Unauthorized：
     - Token是否存在？是否过期？
     - Token是否正确放在请求头中？格式是否正确？（如Bearer token）
     - 登录接口是否正常返回Token？
   - 如果是403 Forbidden：
     - 当前用户是否有权限访问该接口？
     - 是否缺少必要的角色或权限？

4. **后端返回问题**
   - 后端返回的HTTP状态码是多少？
   - 响应体的错误信息是什么？后端是否有具体的错误提示？
   - 是500错误吗？那是后端内部错误，查看后端日志
   - 是502/504吗？可能是网关问题或后端服务无响应
   - 返回的数据结构是否和前端预期一致？字段名、层级是否正确？

5. **网络层问题**
   - 请求是否超时？后端接口响应时间是多少？
   - 是所有接口都失败还是个别接口？
   - 请求是否被广告拦截插件、浏览器安全策略拦截？
   - HTTPS页面是否请求了HTTP接口被混合内容拦截？

### 验证方法
- 复制Network中的请求为curl命令，在终端执行，对比结果
- 在请求拦截器中打印请求配置，确认参数正确
- 在响应拦截器中打印响应/错误信息，查看后端返回详情

## 四、页面渲染与交互问题

### 现象特征
- 页面白屏
- 部分组件不显示
- 样式错乱
- 点击按钮无反应
- 页面卡顿

### 引导式排查

1. **白屏问题**
   - 打开控制台，是否有JavaScript错误？第一个错误是什么？
   - 查看Elements面板，HTML结构是否正常渲染？
   - 路由配置是否正确？访问的路由是否存在？
   - 入口HTML文件中的JS/CSS资源路径是否正确？
   - 是否有JS加载失败？（404或网络错误）
   - 是否是首屏渲染报错导致整个应用挂掉？有没有错误边界（Error Boundary）？

2. **组件不显示/数据不展示**
   - 组件是否被条件渲染（v-if/&&/三元表达式）？条件是什么？
   - 数据是否正确获取？打印从后端返回的数据
   - 数据结构是否和组件预期一致？
   - 列表渲染的key是否正确？
   - 是否有CSS样式导致元素被隐藏（display: none、visibility: hidden、opacity: 0、z-index被覆盖）？
   - 元素是否在视口外？

3. **样式问题**
   - 样式是否正确加载？在Elements中查看Computed样式
   - 选择器优先级问题？样式被其他规则覆盖？
   - CSS单位是否正确？（px/rem/vw/%）
   - 是否是浏览器兼容性问题？某些CSS属性在你使用的浏览器中不支持？
   - 响应式断点是否正确？窗口大小变化时样式是否正常？
   - 全局样式是否污染了组件样式？（CSS Modules/Scoped CSS是否生效）

4. **交互问题**
   - 事件绑定是否正确？方法名是否拼写正确？
   - 按钮/元素是否被禁用或有遮罩层遮挡？
   - 事件触发了吗？在事件处理函数中打印日志确认
   - 事件处理函数中的this指向/Vue Composition API的响应式是否正确？
   - 是否有JS报错导致后续代码不执行？
   - 异步操作（接口请求）是否正确等待结果？

### 验证方法
- 使用Vue DevTools/React DevTools查看组件状态和Props
- 在关键位置添加console.log/debugger，追踪执行流程
- 注释掉部分组件，逐步定位是哪个组件导致问题
- 对比正常页面和异常页面的DOM结构、CSS差异

## 五、鸿蒙（HarmonyOS）编译问题

### 现象特征
- DevEco Studio编译报错
- 签名失败
- 安装到设备失败
- HAP包构建异常
- ArkTS语法错误

### 引导式排查

1. **开发环境验证**
   - DevEco Studio版本是否符合项目要求？
   - SDK版本是否匹配？是否安装了对应版本的SDK？
   - Node.js环境是否正确配置在DevEco Studio中？
   - ohpm（OpenHarmony Package Manager）依赖是否正确安装？
   - 项目目录路径是否有中文或空格？建议使用纯英文路径

2. **签名与证书问题**
   - 是否配置了正确的签名证书？
   - 证书是否过期？
   - 调试证书是否添加了设备UDID？
   - 发布证书和调试证书是否用混了？
   - 是否登录了华为开发者账号？账号是否有权限？

3. **编译构建错误**
   - 具体的编译错误信息是什么？定位到哪个文件哪一行？
   - ArkTS语法是否符合规范？是否使用了不支持的TS语法？
   - 资源文件（图片、字符串、布局）引用是否正确？资源ID是否存在？
   - module.json5/config.json配置是否正确？
   - 权限配置是否完整？
   - 依赖的HAR/HSP包是否正确引入？版本是否兼容？
   - 构建缓存是否有问题？尝试Clean Project后重新构建

4. **设备安装问题**
   - 设备是否开启开发者模式和USB调试？
   - 设备和电脑是否正常连接？hdc工具能否识别设备？运行`hdc list targets`
   - 设备系统版本是否兼容应用的API版本？
   - 设备存储是否充足？
   - 之前是否安装过同应用不同签名的版本？需要先卸载
   - 安装过程的错误码是什么？根据错误码查找对应问题

5. **应用运行问题**
   - 应用启动时是否闪退？查看HiLog日志
   - 网络请求是否正常？是否配置了网络权限和明文传输配置？
   - WebView加载是否有问题？
   - 权限是否动态申请？（危险权限需要用户授权）
   - 页面路由是否正确配置？

### 验证方法
- 查看DevEco Studio的Build Output窗口，查看完整的编译日志
- 使用hdc工具查看设备日志：`hdc hilog`
- 尝试创建一个新的Hello World项目，看是否能正常编译运行，排除环境问题
- 查看HarmonyOS官方文档对应错误码的含义
