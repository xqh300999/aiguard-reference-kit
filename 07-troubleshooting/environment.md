# 环境问题排查

环境问题是最常见的故障根源，包括依赖安装失败、版本不匹配、路径配置错误等。遇到问题时，建议首先排查环境层。

## 诊断前自查清单

在深入排查前，请先回答以下问题：
- 你使用的操作系统是什么？（Windows/macOS/Linux）版本是多少？
- 你是否严格按照项目文档中的步骤进行环境配置？
- 最近是否修改过系统环境变量或项目配置？
- 是否在虚拟环境/conda环境中操作？环境是否正确激活？
- 项目依赖是否完整安装？

---

## 一、依赖安装问题

### 现象特征
- `pip install`/`npm install` 报错
- 安装过程中出现编译错误
- 提示权限不足
- 下载速度慢或超时

### 引导式排查

1. **Python依赖问题**
   - 你当前的Python版本是多少？运行 `python --version` 或 `python3 --version`
   - 项目要求的Python版本范围是什么？两者是否匹配？
   - 是否使用了虚拟环境？虚拟环境是否已激活？
   - pip版本是否过旧？尝试升级pip：`pip install --upgrade pip`
   - 错误信息中提到的是哪个包安装失败？
   - 该包是否有系统级依赖需要先安装？（如某些包需要gcc、python3-dev等）
   - 是否使用了国内镜像源？网络是否能正常访问PyPI？

2. **Node.js依赖问题**
   - 你当前的Node.js版本是多少？运行 `node --version`
   - npm/yarn/pnpm版本是多少？
   - 项目要求的Node.js版本范围是什么？
   - 是否存在node_modules目录和package-lock.json/yarn.lock不一致的情况？
   - 错误日志中提示的是哪个包安装失败？
   - 是否需要配置npm镜像源？

3. **权限问题**
   - 错误信息中是否包含 "Permission denied" 或 "EACCES"？
   - 你是否在系统级Python/Node.js环境中安装包？
   - 是否应该使用虚拟环境而不是全局安装？
   - 项目目录的读写权限是否正确？

### 验证方法
- 创建一个全新的虚拟环境，重新安装依赖，看是否复现问题
- 检查 `requirements.txt`/`package.json` 中是否有版本冲突
- 查看错误日志的完整输出，定位第一个报错的位置

## 二、版本不匹配问题

### 现象特征
- 启动时提示模块找不到，但实际已安装
- 出现 `ImportError`、`ModuleNotFoundError`
- API调用参数不匹配
- 运行时出现意外的AttributeError或TypeError

### 引导式排查

1. **运行时版本验证**
   - 执行 `which python` / `which node`，确认使用的是正确的可执行文件路径
   - 在项目代码中打印 `sys.version`（Python）或 `process.version`（Node.js）
   - 确认IDE/编辑器使用的解释器版本是否与终端一致
   - 是否存在多个Python/Node.js版本共存的情况？（如pyenv、nvm管理的版本）

2. **依赖版本验证**
   - 运行 `pip list` / `npm list` 查看已安装包的实际版本
   - 与 `requirements.txt`/`package.json` 中指定的版本对比
   - 是否有包版本超出了兼容范围？
   - 是否存在依赖包的主版本号不兼容？（主版本号变更通常意味着不兼容的API改动）

3. **配置与代码版本**
   - 代码是否是最新版本？运行 `git status` 检查是否有未提交的变更
   - 配置文件是否与当前代码版本匹配？
   - 是否切换过分支但未重新安装依赖？

### 验证方法
- 使用 `pip freeze > installed.txt` 导出当前依赖，与requirements.txt做对比
- 检查是否存在重复安装的包（不同版本）
- 尝试在全新环境中按照指定版本安装，看是否正常

## 三、路径问题

### 现象特征
- 提示文件或目录不存在
- 相对路径引用失败
- 配置文件找不到
- 日志输出、文件读写异常

### 引导式排查

1. **工作目录问题**
   - 你当前在哪个目录下执行命令？运行 `pwd` 确认
   - 项目期望的工作目录是哪里？
   - 代码中是否使用了相对路径？相对路径是相对于哪个目录解析的？
   - 启动脚本是否正确切换了工作目录？

2. **环境变量与路径配置**
   - 相关的环境变量是否已正确设置？（如PATH、PYTHONPATH、NODE_PATH等）
   - 运行 `echo $PATH`（Linux/macOS）或 `echo %PATH%`（Windows）检查PATH配置
   - 配置文件中的路径是绝对路径还是相对路径？
   - 路径中是否包含空格、中文或特殊字符？这可能导致某些工具解析错误
   - Windows系统下路径分隔符是否正确？（\ vs /）

3. **文件权限与存在性**
   - 程序尝试访问的文件/目录是否真的存在？手动检查一下
   - 运行程序的用户是否有该文件/目录的读写权限？
   - 文件扩展名是否被隐藏导致看错？（如.txt实际是.txt.txt）

### 验证方法
- 在代码中添加打印语句，输出关键路径的绝对路径
- 使用Python的 `os.path.exists()` 或Node.js的 `fs.existsSync()` 验证路径存在性
- 使用绝对路径临时替换相对路径，看问题是否解决

## 四、操作系统特有问题

### 引导式排查

1. **Windows系统**
   - 是否正确配置了环境变量（如Python、Node.js加入PATH）？
   - 是否使用了PowerShell还是CMD？某些命令在不同shell中有差异
   - 是否遇到路径长度限制问题？（MAX_PATH限制）
   - 杀毒软件是否拦截了某些文件操作或网络访问？
   - WSL环境与Windows原生环境是否混淆？

2. **macOS系统**
   - 是否安装了Xcode Command Line Tools？运行 `xcode-select --install`
   - Homebrew安装的依赖是否在PATH中？
   - M1/M2芯片的Mac是否遇到了架构兼容性问题？（arm64 vs x86_64）
   - 安全设置是否阻止了未签名应用的运行？

3. **Linux系统**
   - 是否安装了必要的编译工具链？（build-essential、python3-dev等）
   - 发行版的包管理器是否安装了系统级依赖？
   - 是否使用了正确的用户权限？
   - SELinux/AppArmor是否阻止了某些操作？

### 通用建议
- 记录下你所做的每一步修改和结果，便于回溯
- 不要随意升级不相关的包，这可能引入新的问题
- 如果某个步骤卡住，尝试搜索具体的错误信息，而不是笼统搜索"安装失败"
