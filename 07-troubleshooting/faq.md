# 常见问题FAQ

本文档汇总项目开发过程中常见的问题及解答思路。如果在这里没有找到答案，再按照分层排查指南逐步诊断。

---

## 环境相关

**Q: pip install安装依赖很慢，经常超时怎么办？**

可以尝试：
1. 使用国内PyPI镜像源，如清华源、阿里云源
2. 升级pip到最新版本
3. 检查网络连接，必要时配置代理
4. 对于需要编译的包，确认系统已安装必要的编译工具链

**Q: 安装依赖时提示权限错误Permission denied怎么办？**

不要使用sudo pip install！这会污染系统Python环境。正确做法：
1. 使用虚拟环境（venv/virtualenv/conda）
2. 检查项目目录权限是否正确
3. 如果是全局安装的包，使用`--user`参数安装到用户目录

**Q: 明明安装了包，但运行时提示ModuleNotFoundError？**

可能原因：
1. 安装包和运行代码使用的不是同一个Python环境，检查解释器路径
2. IDE配置的Python解释器和终端不一致
3. 包名和导入名可能不同（如pip install python-dateutil，但import dateutil）
4. 虚拟环境没有正确激活

**Q: Windows/macOS/Linux下路径有什么需要注意的？**

1. Windows用反斜杠`\`，但在代码中建议统一用正斜杠`/`或使用os.path处理
2. 避免路径中包含中文、空格、特殊字符
3. 不要硬编码绝对路径，使用相对路径或配置文件
4. Windows下注意盘符大小写和路径长度限制（MAX_PATH）

## 后端相关

**Q: 后端启动提示端口被占用怎么办？**

1. 先确认哪个进程占用了端口：`lsof -i :端口号`（macOS/Linux）或`netstat -ano | findstr :端口号`（Windows）
2. 如果是之前没正常退出的服务，结束该进程
3. 如果是其他必要程序占用，修改后端配置使用其他端口

**Q: Home Assistant连接失败，返回401 Unauthorized？**

1. 确认使用的是Long-Lived Access Token，而不是临时的API密码
2. Token是否正确生成（在HA用户页面→创建令牌）
3. Token是否过期或被删除
4. 请求头格式是否正确：`Authorization: Bearer YOUR_TOKEN`
5. 确认HA没有配置IP白名单限制

**Q: MQTT连接成功但收不到消息？**

1. 确认订阅的主题是否正确，注意MQTT主题是大小写敏感的
2. 检查QoS级别是否匹配
3. 确认发布者确实在向该主题发布消息，可以用mosquitto_sub命令行工具验证
4. Broker是否配置了ACL权限控制
5. Client ID是否重复导致被踢下线

**Q: SQLite数据库被锁定怎么办？**

1. 检查是否有多个进程同时写入数据库
2. 确认数据库连接正确关闭，没有连接泄漏
3. 考虑使用WAL模式提高并发
4. 如果并发写入需求高，考虑切换到MySQL/PostgreSQL

## 前端相关

**Q: npm install速度很慢怎么办？**

1. 使用国内npm镜像源（淘宝npmmirror镜像）
2. 如果是特定包下载慢，可以配置该包的单独镜像
3. 尝试使用yarn或pnpm替代npm
4. 清除npm缓存：`npm cache clean --force`

**Q: 前端开发时API请求跨域怎么办？**

开发环境解决方案：
1. 配置开发服务器代理（vite proxy/webpack devServer proxy）
2. 后端配置CORS允许开发服务器地址
3. 不要用浏览器插件禁用CORS安全策略来调试，这会掩盖真实问题

生产环境解决方案：
1. 使用Nginx等反向代理，将前端和API部署在同源下
2. 后端正确配置CORS允许生产环境域名

**Q: 页面白屏但没有任何报错？**

1. 检查是否JS/CSS资源加载失败（404）
2. 检查路由是否正确配置，访问的路径是否存在
3. 是否有根组件渲染错误但被错误边界吞掉了
4. 尝试在入口文件最开头加try-catch和日志输出
5. 检查HTML文件中资源引用路径是否正确（部署到子目录时要特别注意）

**Q: 鸿蒙DevEco Studio编译失败提示SDK不匹配？**

1. 检查项目指定的SDK版本是否已安装（Tools→SDK Manager）
2. 确认compileSdkVersion、compatibleSdkVersion等配置正确
3. 尝试Sync Project with Gradle Files
4. 检查ohpm依赖是否正确安装，执行ohpm install
5. 项目路径不要有中文和空格

**Q: 鸿蒙应用安装到设备失败？**

1. 设备是否开启开发者模式和USB调试
2. 运行`hdc list targets`确认设备被识别
3. 如果提示签名错误，重新配置签名证书，确保设备UDID已添加
4. 如果之前安装过，先卸载旧版本（不同签名不能覆盖安装）
5. 确认设备系统版本不低于应用兼容版本

## 网络相关

**Q: 本机可以访问后端，其他机器访问不了？**

大概率是以下原因：
1. 后端只绑定了127.0.0.1，需要绑定0.0.0.0才能外部访问
2. 操作系统防火墙没有放通后端端口
3. 如果是云服务器，检查安全组/网络ACL配置
4. 两台机器不在同一局域网，或者有AP隔离

**Q: WebSocket连接成功但很快自动断开？**

1. 检查是否配置了心跳（ping/pong），很多防火墙会断开空闲连接
2. 查看后端日志，断开时是否有错误
3. Nginx代理的话，检查proxy_read_timeout配置是否足够大
4. 前端是否有异常重连逻辑，确认断开的错误码是什么
5. 是否有多个标签页打开，Client ID或会话冲突

**Q: IoT设备连接WiFi后无法连接服务器？**

1. 确认设备获取到了正确的IP地址、网关、DNS
2. 检查路由器是否开启AP隔离，禁止设备间通信
3. 很多设备只支持2.4GHz WiFi，确认连接的不是5GHz
4. 设备上配置的服务器地址是否正确（使用IP不要用localhost）
5. 确认服务器端口在局域网内可访问，先用电脑测试端口连通性

## 硬件相关

**Q: 电脑识别不到ESP32开发板的串口？**

1. 确认使用的是数据线，不是只有充电功能的线
2. 安装对应USB转串口芯片的驱动（CH340/CP2102/FT232）
3. 换一个USB口试试，尤其是台式机用后置USB口
4. 在设备管理器中查看是否有未识别的设备
5. Linux/macOS下检查用户是否有串口访问权限（dialout组）

**Q: ESP32刷写固件时一直等待连接？**

1. 需要手动进入下载模式：按住BOOT键不放，点击烧录，看到开始连接后松开BOOT键
2. 有些板子需要在点击烧录后按一下RESET键
3. 降低烧录波特率（从921600改成115200）
4. 检查串口是否被其他程序（如串口监视器）占用
5. 确认选择了正确的串口号

**Q: 传感器没有数据输出？**

排查顺序：
1. 先检查供电：VCC和GND有没有接反？电压对不对？（3.3V vs 5V）
2. 检查通信引脚：SDA/SCL、TX/RX有没有接反？
3. I2C设备运行I2C扫描程序，看能否扫描到设备地址
4. 确认代码中引脚定义和实际接线一致
5. 检查传感器是否有地址选择引脚，地址是否和代码匹配
6. 用另一个同型号传感器交叉验证

**Q: 传感器数据跳变、不准怎么办？**

1. 检查电源是否稳定，增加滤波电容
2. 信号线远离干扰源（电机、电源线）
3. 软件上做滤波处理（滑动平均、中值滤波、卡尔曼滤波）
4. 确认传感器校准是否正确
5. 检查数据转换公式是否正确，单位是否统一

## 调试技巧

**Q: 遇到问题不知道从哪里开始排查怎么办？**

1. 先回忆最近修改了什么，问题从什么时候开始出现
2. 查看完整的错误日志，尤其是第一个错误位置
3. 按照环境→网络→后端→前端→硬件的分层顺序，从下往上验证
4. 简化问题：去掉非必要组件，看最简情况下能否复现
5. 使用二分法：回退部分代码/配置，定位引入问题的变更

**Q: 问题不能稳定复现，偶发性出现怎么办？**

1. 增加更详细的日志，记录关键状态和变量
2. 观察问题出现的规律：特定操作后？高负载时？特定时间？
3. 检查资源泄漏：内存、文件句柄、数据库连接
4. 检查并发问题：竞态条件、死锁、资源竞争
5. 检查网络波动、硬件接触不良等间歇性问题

**Q: 如何快速判断是前端问题还是后端问题？**

1. 打开浏览器Network标签，看API请求：
   - 请求没发出去：前端问题
   - 请求发出去了返回4xx/5xx：看错误码，可能是前端参数不对也可能是后端错误
   - 请求返回200但数据不对：后端返回数据有问题
   - 请求返回200但前端展示不对：前端处理问题
2. 用curl/Postman直接调用API，如果curl也有问题是后端问题
3. 对比正常环境和异常环境的差异

**Q: 串口日志应该看什么？**

1. 设备启动过程的日志：看初始化是否成功，有没有报错
2. WiFi连接日志：看是否成功连接，获取到IP没有
3. MQTT连接日志：是否连接Broker成功
4. 传感器初始化和读取日志：是否初始化成功，是否有读取错误
5. 错误堆栈和异常信息：定位代码错误位置

---

**最后建议**：
- 遇到问题先看错误信息，不要急着改代码
- 搜索错误信息时，用引号引起来精确搜索
- 先理解问题，再尝试解决，不要盲目试错
- 记录排查过程：什么现象、尝试了什么、结果如何
- 如果一个方法试了两次都不行，停下来换个思路，不要重复无效操作
